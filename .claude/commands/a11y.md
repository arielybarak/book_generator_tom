# /a11y — accessibility gate

Accessibility IS the product — TOM serves blind children and their parents. A11y regressions
are product regressions. This gate runs axe-core against every route in both languages and
asserts the "blind-parent path" works keyboard-only. Wire into `/preflight` once it passes.

**Prerequisite:** `/web-verify` set up (headless Chromium installed). Also:
```bash
cd web && npm install --save-dev @axe-core/playwright
```

**$ARGUMENTS** = `[--route PATH]` (default: all routes)

## Axe scan — all routes × both languages

```js
// web/tools/a11y-check.mjs
import { chromium } from 'playwright';
import AxeBuilder from '@axe-core/playwright';

const BASE = process.env.BASE_URL ?? 'http://localhost:5173';
const ROUTES = ['/', '/create', '/preview', '/download'];
const LANGS = ['he', 'en'];
let failed = 0;

const browser = await chromium.launch({ headless: true });

for (const lang of LANGS) {
  for (const route of ROUTES) {
    const page = await browser.newPage();
    await page.goto(`${BASE}${route}?lang=${lang}`);
    await page.waitForLoadState('networkidle');

    const results = await new AxeBuilder({ page })
      .withTags(['wcag2a', 'wcag2aa', 'wcag21aa'])
      .analyze();

    if (results.violations.length) {
      console.error(`[FAIL] ${lang} ${route}: ${results.violations.length} violation(s)`);
      for (const v of results.violations) {
        console.error(`  ${v.id}: ${v.description}`);
        for (const n of v.nodes.slice(0,3))
          console.error(`    → ${n.target.join(', ')}: ${n.failureSummary}`);
      }
      failed++;
    } else {
      console.log(`[OK] ${lang} ${route}`);
    }
    await page.close();
  }
}

// TOM-specific: image alt on generated illustrations
// (checked post-generate via /gen-probe output page, not here — too expensive)

await browser.close();
if (failed) process.exit(1);
```

## Blind-parent path — keyboard-only assertion

Assert the full create→download flow is operable with **zero pointer events**:

```js
const page = await browser.newPage();
await page.goto(`${BASE}/?lang=he`);

// Tab through all interactive elements
let steps = 0;
while (steps < 30) {
  await page.keyboard.press('Tab');
  const el = await page.evaluate(() => document.activeElement?.tagName + ':' + document.activeElement?.getAttribute('aria-label'));
  console.log(`  focus: ${el}`);
  steps++;
  if (el.includes('download') || el.includes('הורדה')) break;
}
// Assert we reached the download step via keyboard alone
if (steps >= 30) {
  console.error('[FAIL] blind-parent path: could not reach download via keyboard in 30 tabs');
  process.exit(1);
}
console.log('[OK] blind-parent path: download reachable keyboard-only');
```

## What it checks

| Check | Threshold |
|---|---|
| WCAG 2.1 AA violations | 0 per route |
| Focus rings visible | Required (axe `focus-visible`) |
| ARIA labels on inputs | Required |
| Color contrast | ≥ AA (4.5:1 text, 3:1 UI) |
| Landmark roles | `main`, `nav` present |
| `prefers-reduced-motion` | Framer MotionConfig enforces it (static check) |
| Blind-parent keyboard path | Full flow in ≤ 30 tabs |

Report: route-by-route table, violation count, and the blind-parent path result. Any failure
blocks deploy (add to `/preflight --web` once this gate consistently passes).
