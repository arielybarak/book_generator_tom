# TOM — Tactile Hebrew Book Generator

A pipeline for creating **3D-printable tactile books** in Hebrew. Each page combines:

- An **AI-generated line-art image** (Stable Diffusion)
- The **Hebrew word** for that image
- The **Braille transliteration** of the word

All three elements are exported as DXF vectors and then merged into a single **STL file** ready for a 3D printer.

---

## What does it produce?

For each book page you get:

| Output | Description |
|---|---|
| `image.png` | Line-art illustration generated by Stable Diffusion |
| `text.png` | Hebrew word, rendered as a clean image |
| `braille.png` | Braille characters for the word |
| `image.dxf` / `text.dxf` / `braille.dxf` | Vector versions of each |
| `page_N.stl` | A single 3D model combining all three layers on a base plate |

The STL has:
- Raised **text** (solid extrusion)
- **Braille domes** (hemisphere bumps, tactile standard)
- Raised **image strokes** (line-art as extruded ridges)

### Key modules at a glance

**`language_funcs.py`** — Hebrew language utilities
- Translates Hebrew → English (for the image generation prompt)
- Interactively adds nikud (vowel marks: dagesh, holam, shuruk, hirik, shin dot)
- Converts Hebrew text (with nikud) to Braille unicode characters

**`image_generator.py`** — Per-page image pipeline
- Loads `segmind/SSD-1B` (Stable Diffusion) via HuggingFace Diffusers
- Generates a clean line-art image from the Hebrew description
- Edge-detects, centers the object, and saves three PNGs: image / text / braille
- Calls `images_to_dxf()` to convert all three to DXF

**`image_funcs.py`** — Low-level image/DXF utilities
- `image_to_dxf_exact()` — converts a processed edge image to an accurate DXF polyline
- `png_to_dxf()` — converts any PNG to a DXF using contour extraction
- `plot_dxf()` — quick matplotlib preview of a DXF file

**`dxf_3d.py`** — 3D model builder (the most complex module)
- Reads three separate DXFs: text shapes, braille circles, image centerlines
- Centers all content on a 150×150mm base plate
- Extrudes text as solid shapes, braille as spherical domes, image as stroked ridges
- Exports to STL (and optionally STEP) using CadQuery

**`flow_manager.py`** — Multi-page orchestrator
- Takes a list of page specs (description, classification, generate flag)
- Creates a timestamped output folder per book
- Processes each page atomically: if a step fails, that page is skipped cleanly
- Returns a `FlowResult` with image locations and STL file paths

**`app.py`** — Gradio web interface
- **Interactive Wizard tab**: step-by-step book builder
  - Enter book title → add pages one at a time
  - Per-page: Hebrew text + visual description + object class
  - Automatic **disambiguation UI** for ambiguous letters (ש/שׂ, ב/בּ, ו/וֹ/וּ, etc.)
  - Download final book as a ZIP of STL files
- **Advanced JSON tab**: paste a full JSON config and generate in one shot

## How to run

### Option 1 — Gradio Web UI (recommended)

```bash
pip install gradio trimesh
python app.py
```

Open the local URL shown in the terminal. Build your book page by page.

### Option 2 — Single page CLI

```bash
# requires: torch, diffusers, cv2, deep_translator, ezdxf, matplotlib
python main.py
```

You'll be prompted in Hebrew to enter an image description and category.

### Option 3 — Programmatic (FlowManager)

```python
from flow_manager import FlowManager

pages = [
    {
        "page_number": 1,
        "image_description": "תפוח",
        "image_classification": "פרי",
        "generate_picture": True,
        "done": False,
    }
]

fm = FlowManager(book_name="my_book", pages=pages)
result = fm.run()
```

### Option 4 — Convert DXFs to STL directly

```bash
python dxf_3d.py --text text.dxf --braille braille.dxf --image image.dxf -o page1.stl
```

## Dependencies

| Package | Purpose |
|---|---|
| `torch` + `diffusers` | Stable Diffusion image generation |
| `opencv-python` (`cv2`) | Image processing, edge detection |
| `ezdxf` | DXF read/write |
| `cadquery` | 3D solid modeling → STL export |
| `pyclipper` | Polygon offsetting (stroke generation) |
| `trimesh` | Alternative STL mesh builder (used in app.py) |
| `gradio` | Web UI |
| `deep_translator` | Hebrew → English translation |
| `matplotlib` | Rendering text/braille images and DXF preview |
| `Pillow` | Image utilities |

For Colab, see `app_colab.ipynb` which includes install commands at the top.

The `NotoSansSymbols2-Regular.ttf` font is required for rendering Braille characters:
```bash
wget -O NotoSansSymbols2-Regular.ttf \
  https://github.com/googlefonts/noto-fonts/raw/main/hinted/ttf/NotoSansSymbols2/NotoSansSymbols2-Regular.ttf
```

## Known issues / work in progress

- `flow_manager.py` has a bug: `os.path.exist()` should be `os.path.exists(path)` — pages with images will silently store `None` locations
- `dxf_3d.py`'s `create_one_page_stl_from_dxf()` still references `args` (CLI args) instead of the function parameters — it needs refactoring to work as an importable function
- `language_funcs.py`'s `add_nikud()` uses interactive `input()` calls, which doesn't work in the Gradio UI — `app.py` handles disambiguation separately via its own UI widgets
